Supervised Textual Relevancy - CMS Rule Matching


Goal:

Given a predefined set V of 'definition' texts and a set K of unstructured 'comment' texts where |K| >> |V|, segment each comment k ∈ K and match the segments to their most relevant definition v ∈ V. A successful tool will split documents coherently, and match the relevant subsections to their corresponding rule. Useful topics include text classification, text embeddings, semantic relevancy, ontology generation, and clustering algorithms.

Because a particular similarity metric does not entail predictability or semantic usefulness, a successful tool must learn semantically useful features to match on. The defining metrics for this tool's success will be the precision and recall over the labelled document set, the training volume required, and the time to classification.

Specifications:

  1. This tool should return relevancy scores over all definitions
  2. The targeted definition size is 300-3000 characters
  3. The targeted comment size is 50-5000 characters
  4. All code should be uploaded to the project github
  5. Work will be done using the public CMS Rules and Comments
  6. Formatted rules and comments, annotations, and ontologies can be found here
  7. Open source libraries and code reuse is OK, provided it is properly attributed
