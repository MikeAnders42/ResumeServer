Automated Onotology Generation


Goal:

Create an algorithm to identify topically salient entities within a document corpus and organize them according to a predefined set of semantic relations: hyper/hyponymy, syn/antonymy, and related-to.

Because "topically salient" is not well-defined, it will be approximated using expert-curated ontologies. Entity relations will likewise be evaluated against these ontologies, as well as manually reviewed by subject matter experts.

Specifications:

  1. Generated ontologies should be in rdf/xml format
  2. Policy corpuses will be ~100-10000 pages in length
  3. All code should be uploaded to the project github
  4. Work will be done using the public DoD Issuances and CMS Manuals
  5. Formatted ontologies and corpuses can be found here
  6. Open source libraries and code reuse is OK, provided it is properly attributed
